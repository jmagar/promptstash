# Documentation Directory

This directory contains all organized documentation for the PromptStash project.

## Directory Structure

### `/guides`

Step-by-step guides and tutorials for getting started and using the application:

- `QUICKSTART.md` - Quick setup and installation guide
- `DEMO.md` - Feature demonstration and walkthrough
- `DATABASE_SETUP.md` - Database configuration and setup
- `TESTING_GUIDE.md` - Testing guidelines and procedures

### `/reference`

Reference documentation and API specifications:

- `API.md` - Complete API endpoint documentation
- `PERFORMANCE_QUICK_REFERENCE.md` - Performance optimization quick reference
- `PORT_CONFIGURATION.md` - Port configuration reference
- `QUICK-REFERENCE.md` - General quick reference
- `SKILLS-README.md` - Skills system reference
- `validation-rules-reference.md` - File validation rules

### `/architecture`

Architecture decisions, design patterns, and technical documentation:

- `CLAUDE.md` - Main coding guidelines for Claude Code
- `PERFORMANCE_OPTIMIZATIONS.md` - Performance optimization strategies
- `SECURITY.md` - Security architecture and best practices
- `TESTING_SUMMARY.md` - Testing architecture and summary
- `UX_IMPROVEMENTS.md` - UX design decisions and improvements

### `/demos`

Demo files, mockups, and example content:

- `promptstash-ui-mockup.html` - UI mockup
- `promptstash.md` - General project description

### `/archived`

Historical documentation (currently empty):

- For superseded documentation that has been replaced
- For deprecated features that were removed
- Not for AI session notes (those belong in `.docs/`)

### `/development`

Active development documentation (empty - for future use)

### `/observability`

Monitoring, logging, and observability documentation:

- `OBSERVABILITY.md` - Main observability guide
- `OBSERVABILITY_QUICK_REFERENCE.md` - Quick reference
- `OBSERVABILITY_SETUP.md` - Setup instructions

## Root Documentation Files

The following documentation files remain in the project root for easy access:

- `README.md` - Project overview and main entry point
- `commitlint.config.js` - Commit message linting configuration
- `eslint.config.js` - ESLint configuration
- `turbo.json` - Turborepo configuration

## Other Documentation Locations

- `.docs/` - **AI Chat Session Archive** - All AI session summaries, implementation notes, and working documentation (NOT for published docs)
- `.claude/` - Claude Code configuration files (agents, skills, commands, hooks)
